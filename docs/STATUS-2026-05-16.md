## 三、当前核心问题：前端交互全面失效

### 问题表现

- 论文阅读页：选择文件按钮无效，拖拽文件无效
- 刷题页：选择文件弹两次窗口，导入无效，模式切换无效，开始刷题无效
- 设置页：API Key保存按钮无效，填了Key仍显示"未配置"，MFAW参数保存无效
- 所有页面的交互按钮点击无任何反应

### 已排查项

1. ✅ 所有JS文件语法正确（`node --check` 全部通过）
2. ✅ 所有文件HTTP 200可访问
3. ✅ MIME类型正确（`application/javascript`）
4. ✅ 后端API正常
5. ✅ 模块自实例化已添加（`new ReaderCore()`, `new SettingsCore(null)`, `new QuizCore()`）
6. ✅ 路由已加模块加载失败可见错误条
7. ✅ app.js已加启动try-catch
8. ✅ entrance.js已加null守卫

### 根因分析（推测）

最可能的原因是**浏览器端ES模块导入链在某个环节静默失败**。具体可能：

1. **`crypto.js` 使用 `crypto.subtle`**：Web Crypto API需要安全上下文（HTTPS或localhost），虽然localhost应支持，但某些浏览器/设置可能阻止。如果`crypto.subtle`不可用，`encryptApiKey`会抛异常→`settings-core.js`导入链断裂。

2. **`api-keys.js` 顶层调用**：可能在模块加载阶段访问了不存在的DOM或API。

3. **reader-core.js 的 `AIPanel` 实例化失败**：AIPanel的`_buildDOM()`方法会重写容器innerHTML，可能与reader.html的DOM结构冲突。

4. **quiz-core.js 的 `loadQuestions` 方法被覆盖**：`_bindQuizHomeEvents`中做了`const origLoad = this.loadQuestions.bind(this); this.loadQuestions = async (file) => {...}`，如果`quiz-core.js`的`loadQuestions`定义有问题，会导致整个quiz模块异常。

5. **模块间循环依赖**：检查是否有import循环。

### 明天排查步骤

1. 打开浏览器 → F12 → Console → 查看红色报错
2. 如果没有任何报错，在Console输入：`window.__lt` 检查全局对象是否存在
3. 输入：`window.__lt.router.currentRoute` 查看当前路由
4. 手动导航：`window.__lt.router.navigate('settings')` → 查看是否报错
5. 检查DOM元素存在性：`document.getElementById('apiKeyDeepSeekSave')`
6. 打开 `http://localhost:3000/diag.html`（如果存在）看模块加载诊断
7. 在Network标签查看是否有JS文件加载失败(红色)
